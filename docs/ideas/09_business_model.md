## 1. 수익 구조 — 사용자 무료, 수익은 B2B

> **핵심 원칙**: 사용자에게 돈을 받지 않는다. 입문 장벽을 낮춰 사용자 수를 극대화하고, 거래 수수료와 사업자 구독료로 수익을 확보한다.

### 수익 스트림

| 수익원 | 모델 | 수수료/단가 | 예상 볼륨 (연) |
| 수익원 | 모델 | 수수료/단가 |
|-------|------|-----------|
| 투어·체험 예약 중개 | % 수수료 | 예약금의 5~10% (글로벌 OTA 대비 낮게 책정) |
| 템플스테이 예약 중개 | % 수수료 | 예약금의 5~8% (공익성 고려 최소 마진) |
| 식당 결제 중개 | % 수수료 | 결제금액의 1.0~1.5% (PG 수수료 수준만 부과) |
| 특산품 마켓 중개 | % 수수료 | 판매금액의 7~10% (배송/CS 연동 비용 포함) |
| K-POP 티켓 중개 | 후원/판매 수수료 | 자율 관람료(Tip) 및 현장 굿즈 판매액의 5~8% (대관료 제로화 전략) |
| 셔틀 버스 중개 | 건당 수수료 | 1인당 1,500~3,000원 |
| **사업자 구독 (SaaS)** | 월정액 | 식당: 1.5만원 / 여행사·공연장: 5만원 (초기 진입 장벽 최소화) |
| 커뮤니티 네이티브 광고 | CPC / CPM | 피드 내 자연스러운 스폰서드 콘텐츠 |
| 지역 상점 O2O 쿠폰 | 건당 수수료 | 사용된 할인 쿠폰 당 500원 (타임 퀘스트 연계) |

| 기능 | 사용자 비용 |
|------|----------|

## 2. UX 원칙 — 입력 최소화 & 빠른 처리

### 7가지 설계 원칙

| # | 원칙 | 구현 방법 |
|---|------|---------|
| 1 | **Zero-Keyboard 우선** | 텍스트 입력 → 카드 선택, 슬라이더, 터치로 대체 |
| 2 | **QR 스캔 중심** | 장소, 메뉴, 상품, 티켓 모두 QR 하나로 진입 |
| 3 | **원터치 결제** | 최초 1회 카드 등록 후 생체인증(지문/FaceID)으로 완료 |
| 4 | **자동 감지** | 언어, 위치, 통화 자동 설정 — 사용자 수동 설정 최소화 |
| 5 | **진행 상태 시각화** | 예약·주문 완료까지 단계 표시 (3단계 이하) |
| 6 | **오프라인 대비** | 핵심 정보 (지도, 메뉴, 예약 확인서) 로컬 캐싱 |
| 7 | **맥락 기반 추천** | 현재 위치·시간·이전 행동 기반으로 다음 행동 추천 |

### 핵심 액션별 최대 터치 수 목표

| 액션 | 목표 터치 수 |
|------|-----------|
| 메뉴 번역 보기 (QR 스캔 후) | **2터치** |
| 식당 메뉴 주문 + 결제 | **4터치** |
| 템플스테이 예약 완료 | **5터치** |
| 특산품 구매 + 해외 배송 | **4터치** |
| 병원 예약 | **5터치** |
| 여행사 상담 시작 | **3터치** |
| K-POP 공연 티켓 구매 | **4터치** |
| 축제 셔틀 버스 예약 | **4터치** |
