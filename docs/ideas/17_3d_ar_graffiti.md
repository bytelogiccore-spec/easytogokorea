# 17. 3D AR 낙서 (3D AR Graffiti Wall) 🎨

> **모토**: *"여행자의 발자취를 공간에 새기다 — GPS 기반 3D 디지털 낙서"*
> **최종 수정**: 2026-03-09

---

## 🧭 1. 컨셉 (Concept)

전 세계 관광지에서 흔히 볼 수 있는 "여기 왔다 감!" 문화를 **건물을 훼손하지 않는 디지털 AR 낙서**로 승화시킵니다.

- **어디서든 허공에 낙서**: 스마트폰 카메라를 통해 실제 공간(건물 벽, 다리, 골목, 하늘)에 3D 텍스트·이모지·그림을 띄울 수 있습니다.
- **추천이 곧 크기**: 다른 여행객이 ❤️ 추천(공감)을 누를수록 **글자/오브젝트가 점점 커지고 화려해집니다**. 누구든 한 번쯤 써본 낙서가 수천 추천을 받아 건물만큼 커지는 장관을 목격할 수 있습니다.
- **GPS + 공간 앵커링**: Apple ARKit / Google ARCore의 **Cloud Anchor** 및 **Geospatial API**를 활용하여, 낙서가 정확한 GPS 좌표 + 3D 공간 좌표에 고정됩니다. 다른 날, 다른 사람이 같은 장소에 오면 정확히 같은 위치에 낙서를 볼 수 있습니다.

## 🔥 2. 핵심 기능 (Core Features)

### 📝 낙서 작성 (Create Graffiti)

| 기능 | 설명 |
|------|------|
| **3D 텍스트** | 허공에 자유롭게 텍스트 입력, 폰트·색상·크기 선택 |
| **3D 이모지/스티커** | 한국 테마 스티커 (🇰🇷 태극기, 🍜 음식, 🏯 건축물, K-POP 아이콘 등) |
| **프리핸드 드로잉** | 손가락으로 허공에 3D 선을 그려 자유로운 그림 가능 |
| **사진/미니영상 부착** | AR 공간에 사진 프레임이나 짧은 영상 클립을 띄워 놓기 |
| **SNS 링크 공유** | 낙서에 Instagram·TikTok·YouTube 등 링크 부착 → 터치 시 프로필/게시물로 이동. 여행 인플루언서·현지 크리에이터가 자신의 콘텐츠를 현장에 AR로 남길 수 있음 |
| **자동 번역 태그** | 작성 언어 자동 감지 → 다국어 번역 태그 자동 부착 |

### ❤️ 추천 & 크기 시스템 (Vote-to-Scale)

```
추천수에 따른 낙서 크기 변화 (최대 5m 제한)

  0~9 추천    →  ○ 노트 크기 (30cm) / 반투명 / 5m 이내에서만 보임
  10~49 추천   →  ◎ 포스터 크기 (1m) / 살짝 발광 / 20m 이내
  50~199 추천  →  ◉ 현수막 크기 (3m) / 네온 효과 / 50m 이내
  200+ 추천    →  ★ 간판 크기 (5m, MAX) / 파티클 이펙트 / 100m 이내
```

> ⚠️ **왜 5m가 상한인가?**
> - 건물 크기(30m+) 낙서는 AR 화면을 압도해 **다른 낙서가 가려지고**, 사용자 경험이 떨어짐
> - 5m(대형 간판 수준)이면 충분히 눈에 띄면서도 **현실 풍경을 해치지 않는** 적정선
> - 대신 200+ 추천 낙서는 크기 대신 **파티클·발광·사운드 등 이펙트**로 존재감 표현

### AR 과밀 방지 규칙

| 규칙 | 설명 |
|------|------|
| **동시 표시 제한** | 화면에 최대 **20개** 낙서만 표시 (score 상위 순) |
| **최소 간격** | 같은 지점 반경 3m 내에는 **1개만** 표시 (나머지는 "외 N개" 표시) |
| **LOD (Level of Detail)** | 먼 낙서는 아이콘으로, 가까이 가면 풀 렌더링 |
| **사용자 밀도 조절** | 설정에서 "낙서 밀도" 슬라이더 제공 (적게~기본~많이) |

- **시간 감쇠(Time Decay)**: 오래된 낙서는 서서히 투명해지며, 지속적으로 추천을 받아야 유지됩니다.
- **Top 10 주간 명예의 전당**: 매주 가장 많은 추천을 받은 낙서 Top 10을 소셜 피드에 자동 노출합니다.

### 🌍 공간 탐색 (Explore)

| 기능 | 설명 |
|------|------|
| **AR 뷰** | 카메라를 들면 주변 낙서가 3D로 떠오름 (반경 설정 가능) |
| **지도 뷰** | 낙서 핫스팟을 히트맵으로 표시, 밀도 높은 지역 = 낙서가 많은 인기 장소 |

| **테마 필터** | #맛집추천 #인생샷 #꿀팁 #고백 #응원 등 태그 필터 |

## 📍 3. GPS + 공간정보 기술 스택 (Spatial Tech)

| 기술 | 용도 |
|------|------|
| **Google ARCore Geospatial API** | Android — 실제 GPS 좌표 + VPS(Visual Positioning System)로 cm 단위 정밀 배치 |
| **Apple ARKit + Location Anchor** | iOS — GPS 기반 AR 앵커링 |
| **Google Cloud Anchors** | 크로스 플랫폼 — 한 사용자가 만든 앵커를 다른 사용자가 동일하게 볼 수 있게 공유 |
| **Mapbox / Google Maps 3D Tiles** | 지도 뷰의 3D 건물 위에 낙서 위치를 시각화 |
| **GeoHash 인덱싱** | 반경 검색 최적화 — 현재 위치 근처의 낙서만 빠르게 로드 |

### 데이터 모델 (간략)

```
Graffiti {
    id: UUID,
    author_id: UUID,
    content_type: TEXT | EMOJI | DRAWING | PHOTO,
    content_data: JSON,         // 텍스트, 3D 좌표 배열, 이미지 URL 등
    location: {
        lat: f64,
        lng: f64,
        altitude: f64,
        heading: f32,           // 낙서가 바라보는 방향
        cloud_anchor_id: String // ARCore/ARKit 공유 앵커
    },
    votes: i32,                 // 추천 수 → 크기 결정
    scale_level: u8,            // 0~4 (계산된 크기 단계)
    language: String,
    tags: Vec<String>,
    created_at: DateTime,
    decay_factor: f32,          // 시간 감쇠 계수
    is_active: bool             // 감쇠로 사라졌는지 여부
}
```

- **스탬프 랠리 연동**: [12. 성지 스탬프 랠리](12_smart_qr_location.md)의 체크인 지점에서 낙서를 남기면 **보너스 포인트** 지급

## 🧠 5-1. 추천 알고리즘 (Recommendation System)

### 왜 트위터 수준의 ML이 필요 없는가

| 항목 | 트위터 | 우리 3D 낙서 |
|------|--------|-------------|
| 후보군 | 수억 개 → ML로 1,500개 추림 | **GeoHash로 이미 50~200개** (위치 기반) |
| 개인화 | 팔로우·관심사 기반 복잡한 ML | **위치 + 언어**면 충분 |
| 실시간성 | 초단위 피드 갱신 | 낙서는 수일~수주 단위로 변화 |
| 인프라 | GPU 클러스터 | **서버 사이드 SQL 쿼리 한 방** |

### 3단계 파이프라인

```
사용자가 카메라를 들면:

 1단계: 공간 필터링 (GeoHash)
 ─────────────────────────
 현재 위치 반경 내 낙서만 DB에서 로드
 100만 개 → ~50~200개로 축소
 (GeoHash 인덱스 쿼리 한 번이면 끝)

 2단계: 점수 계산 (스코어링)
 ─────────────────────────
 score = (votes^0.8) × decay(age) × lang_bonus × dist_weight

 3단계: 필터링
 ─────────────────────────
 • is_active = false → 미표시
 • 신고 3회 이상 → 숨김
 • 사용자 차단 목록 → 제외
 • 결과를 score 내림차순으로 AR 렌더링
```

### 스코어링 공식

```
score = (votes ^ 0.8) × decay(age) × lang_bonus × dist_weight

votes^0.8
  → 추천 수에 감쇄 지수 적용 (1000추천이 100추천보다 10배가 아닌 ~6.3배)
  → 소수 인기 낙서의 독점 방지

decay(age) = 1 / (1 + age_days / 7)
  → 7일 경과 시 점수 반감
  → 30일 경과 시 ~0.19
  → 지속적으로 추천받지 않으면 자연 소멸

lang_bonus
  → 사용자 언어와 동일하면 1.2, 아니면 1.0
  → 영어 사용자는 영어 낙서를 살짝 우선 노출

dist_weight = 1 / (1 + distance_m / 100)
  → 100m 떨어지면 점수 반감
  → 가까운 낙서가 더 크고 선명하게 보이는 자연스러운 AR 경험
```

### 실제 SQL 쿼리 예시 (PostgreSQL + PostGIS)

```sql
SELECT id, content_type, content_data, votes, created_at,
       ST_Distance(location, ST_MakePoint($lng, $lat)) AS dist_m,
       -- 스코어 계산
       POWER(votes, 0.8)
       * (1.0 / (1 + EXTRACT(DAY FROM NOW() - created_at) / 7.0))
       * CASE WHEN language = $user_lang THEN 1.2 ELSE 1.0 END
       * (1.0 / (1 + ST_Distance(location, ST_MakePoint($lng, $lat)) / 100.0))
       AS score
FROM graffiti
WHERE is_active = true
  AND ST_DWithin(location, ST_MakePoint($lng, $lat), $radius_m)
  AND id NOT IN (SELECT graffiti_id FROM reports GROUP BY graffiti_id HAVING COUNT(*) >= 3)
  AND author_id NOT IN (SELECT blocked_id FROM user_blocks WHERE user_id = $user_id)
ORDER BY score DESC
LIMIT 100;
```

### 향후 확장 (규모 커질 때)

| 단계 | 조건 | 추가할 것 |
|------|------|----------|
| MVP | 10만건 이하 | 위 SQL 쿼리만으로 충분 |

### 🎯 AR 광고 모델 (핵심 수익원)

기존 모바일 배너 광고와 달리, AR 광고는 **현실 공간에 자연스럽게 녹아들어** 거부감이 적고 전환율이 높습니다.

#### 상품 라인업

| 상품 | 대상 | 예시 | 가격 모델 |
|------|------|------|----------|
| **📍 AR 간판** | 소상공인 | 내 식당 앞에 AR 메뉴판 + "오늘 10% 할인" 표시 | SaaS 구독 포함 (월 1.5만~) |
| **🏢 AR 빌보드** | 브랜드/대기업 | 명동·강남 핫스팟에 3D 브랜드 오브젝트 설치 | CPM / 월정액 (50만~) |
| **🎪 AR 이벤트존** | 관광지/지자체 | 특정 장소 방문 시 AR 이벤트 팝업 (예: 해운대 AR 불꽃놀이) | 캠페인 단위 계약 |
| **🎁 AR 쿠폰** | 매장 | 매장 반경 50m 진입 시 AR 쿠폰 팝업 → 터치하면 할인 적용 | CPC (클릭당 과금) |
| **🗺️ AR 경로 광고** | 프랜차이즈 | 일정 플래너 경로 추천 시 "이 길에 ★매장" AR 표시 | CPC / 월정액 |
| **👤 인플루언서 AR** | 크리에이터 | 인플루언서 "여기 맛집!" AR 낙서 + 인스타 링크 → 가게가 비용 부담 | 수익 쉐어 |

#### 기존 광고 vs AR 광고

| 항목 | 모바일 배너 | AR 광고 |
|------|-----------|---------|
| 거부감 | 높음 (닫기 버튼 찾음) | **낮음** (풍경의 일부) |
| CTR | 0.5~1% | **5~15%** (예상, 물리적 근접성) |
| 위치 정밀도 | GPS 반경 km | **반경 m 단위** |
| 전환 측정 | 클릭/노출만 | 클릭 + **실제 방문 전환** 측정 가능 |

#### "광고도 콘텐츠" 메커니즘

광고 낙서는 일반 낙서와 **같은 형태**로 표시되되, "Sponsored" 태그가 붙습니다:

```
일반 낙서:   사용자 작성 → 추천으로 성장 → 시간 감쇠 → 소멸
광고 낙서:   광고주 작성 → 추천으로 성장 → 계약 기간 만료 시 소멸
                          └─ "Sponsored" 태그 표시
                          └─ 사용자가 추천/무시 가능
                          └─ 재미있는 광고는 바이럴, 재미없는 광고는 자연 소멸
```

#### 광고 품질 보호 규칙

| 규칙 | 설명 |
|------|------|
| **비율 제한** | 화면 내 광고 낙서 비율 **최대 20%** (20개 중 4개) |
| **최소 간격** | 광고 낙서 간 **50m 이상** 이격 |
| **빈도 제한** | 같은 광고주 낙서는 사용자당 **하루 3회** 노출 제한 |
| **사용자 제어** | "광고 줄이기" 설정 제공, 특정 광고주 차단 가능 |

| **재방문 유도** | 내 낙서가 얼마나 커졌는지 확인하러 재방문 → 앱 잔존율 상승 |

## 💾 9. 데이터 형태 및 크기 산정 (Data Size Estimation)

### 낙서 유형별 단건 데이터 크기

| 유형 | 저장 데이터 | 단건 크기 | 비고 |
|------|-----------|----------|------|
| **3D 텍스트** | 텍스트(UTF-8) + 폰트ID + 색상 + 위치 메타 | **~0.5 KB** | 가장 가벼움 |
| **이모지/스티커** | 스티커 ID(참조) + 위치·크기·회전 메타 | **~0.3 KB** | 에셋은 앱 번들에 포함 |
| **프리핸드 드로잉** | 3D 좌표 배열(점 200~2000개) + 색상·굵기 | **2~20 KB** | 복잡도에 비례 |
| **사진 부착** | 압축 이미지(WebP) + 위치 메타 | **50~200 KB** | 해상도 제한 필요 (max 1024px) |
| **미니 영상** | 압축 영상(WebM, 5초 이내) + 위치 메타 | **300~800 KB** | 5초·720p 제한 권장 |

### 공통 메타데이터 (모든 낙서에 포함)

```
공통 메타데이터 크기: ~200 Bytes

├─ id (UUID)           : 16B
├─ author_id (UUID)    : 16B
├─ content_type (enum) : 1B
├─ location            : 36B
│  ├─ lat (f64)        : 8B
│  ├─ lng (f64)        : 8B
│  ├─ altitude (f64)   : 8B
│  ├─ heading (f32)    : 4B
│  └─ geohash (str)    : 8B   ← 반경 검색용
├─ cloud_anchor_id     : 40B  ← Google Cloud Anchor 식별자
├─ votes (i32)         : 4B
├─ scale_level (u8)    : 1B
├─ language (str)      : 5B
├─ tags (JSON)         : 50B  (평균 3개 태그)
├─ created_at          : 8B
├─ decay_factor (f32)  : 4B
└─ is_active (bool)    : 1B
```

### 스케일링 시나리오

> 가정: 낙서 유형 비율 — 텍스트 50% / 이모지 20% / 드로잉 15% / 사진 10% / 영상 5%

| 규모 | 낙서 수 | 메타데이터(DB) | 미디어(오브젝트 스토리지) | 합계 |
|------|--------|---------------|----------------------|------|
| **MVP (6개월)** | 10,000 | ~2 MB | ~150 MB | **~152 MB** |
| **성장기 (1년)** | 100,000 | ~20 MB | ~1.5 GB | **~1.5 GB** |
| **활성기 (2년)** | 1,000,000 | ~200 MB | ~15 GB | **~15 GB** |

> **참고 — Cloud Anchor 관련**
> - Google ARCore Cloud Anchor는 **Google 서버에 3D feature map을 저장**하고, 우리는 반환된 `cloud_anchor_id`(문자열)만 DB에 보관
> - Cloud Anchor는 **최대 365일 유지** (OAuth 인증 시), API Key 방식은 24시간
> - 앵커 resolve 제한: **300회/분/IP·프로젝트**
> - 즉, 공간 앵커 자체의 용량은 우리 서버에 부담 없음. **우리가 저장하는 건 메타데이터 + 미디어 파일**

## ☁️ 10. 무료 저장 매체 비교 (Free Storage Options)

### 주요 무료 티어 비교

| 서비스 | 무료 스토리지 | DB 무료 | 이그레스(전송) | 특징 |
|--------|-------------|---------|-------------|------|
| **Cloudflare R2** | **10 GB** | ❌ (별도) | **무제한 무료** 🔥 | Class A 100만, Class B 1000만 ops/월 |
| **Firebase Spark** | 5 GB (Storage) | 1 GB (Firestore) | 10 GB/월 | 통합 Auth·DB·Storage·Functions |
| **Supabase Free** | 1 GB (Storage) | 500 MB (Postgres) | 5 GB/월 | 오픈소스, PostgreSQL 기반 |
| **AWS S3 Free Tier** | 5 GB (12개월) | ❌ | 100 GB/월 (12개월) | 12개월 한정, 이후 유료 |

### 추천 조합 (하이브리드 아키텍처)

```
  ┌─────────────────────────────────────────────┐
  │             추천 무료 아키텍처                  │
  ├─────────────────────────────────────────────┤
  │                                             │
  │  [메타데이터 DB]     [미디어 파일 저장]         │
  │  Supabase Free      Cloudflare R2            │
  │  (PostgreSQL)       (Object Storage)         │
  │  500 MB 무료         10 GB 무료               │
  │                     이그레스 완전 무료          │
  │       │                    │                 │
  │       └─── API Server ────┘                  │
  │            (Rust/Axum)                       │
  │                                             │
  └─────────────────────────────────────────────┘
```

| 역할 | 서비스 | 무료 한도 | MVP 충분 여부 |
|------|--------|----------|-------------|
| **메타데이터 DB** | Supabase (PostgreSQL) | 500 MB | ✅ 10만건 메타 = ~20 MB |
| **미디어 저장** | Cloudflare R2 | 10 GB | ✅ 10만건 미디어 = ~1.5 GB |
| **이미지 CDN** | Cloudflare (기본) | 무제한 | ✅ R2 연동 시 자동 CDN |
| **인증** | Supabase Auth | 50,000 MAU | ✅ MVP 충분 |
| **공간 앵커** | Google Cloud Anchors | 무료 (ARCore 부속) | ✅ 앵커 ID만 DB 저장 |

> **💡 결론**: MVP~성장기(10만건)까지 **완전 무료**로 운영 가능. 활성기(100만건, ~15 GB) 진입 시 R2 유료 전환 필요하나 이그레스 무료이므로 비용 매우 낮음 (스토리지만 $0.015/GB·월 = **~$0.23/월**).
