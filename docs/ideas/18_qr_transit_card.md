# 18. QR 교통카드 (QR Transit Pass) 🚇

> **모토**: *"카드 없이, 충전 없이 — QR 하나로 한국 대중교통 완전 정복"*
> **최종 수정**: 2026-03-09

---

## 🧭 1. 문제 정의 (Problem)

외국인 관광객이 한국 대중교통을 이용할 때 겪는 핵심 불편:

| 문제 | 상세 |
|------|------|
| **물리 카드 구매** | 티머니 카드를 편의점에서 별도 구매해야 함 (카드비 2,500~4,000원) |
| **현금 충전** | 한국 원화 현금으로만 충전 가능 → 환전 필수 |
| **잔액 환불** | 귀국 시 잔액 환불 과정이 번거로움 |
| **분실 위험** | 물리 카드 분실 시 잔액 복구 불가 |
| **정보 부족** | 어떤 카드(티머니/캐시비/기후동행카드)를 사야 하는지 혼란 |

---

## 🔍 2. 현재 한국 대중교통 결제 생태계 (2026년 기준)

### 기존 결제 수단

| 수단 | 대상 | 방식 | 외국인 접근성 |
|------|------|------|-------------|
| **티머니 카드** | 내/외국인 | NFC 태그 | ⭐⭐⭐ 편의점 구매, 현금 충전 |
| **모바일 티머니** | 내국인 위주 | NFC (Android) | ⭐ 한국 계좌 필요 |
| **Apple Pay 티머니** | iPhone 사용자 | NFC | ⭐⭐ 해외카드 충전 2025년말 예정 |
| **WOWPASS** | 외국인 전용 | NFC + 선불 직불 | ⭐⭐⭐ 환전+교통 통합 |
| **기후동행카드** | 서울 한정 | NFC | ⭐⭐ 1일~7일 단기권 有 |
| **코레일패스 플러스** | 외국인 전용 | 물리카드 | ⭐⭐ KTX+대중교통 통합 (2025말~) |
| **제주 QR 버스** | 제주 한정 | QR (제로페이) | ⭐⭐⭐⭐ 해외 QR앱 직접 사용 |

### 🔥 핵심 인사이트: 제주 QR 버스 모델

> 2024년 8월부터 제주도 버스에 **제로페이 QR 결제**가 도입되어, **9개국 14개 해외 결제 앱**(알리페이, 위챗페이, GrabPay, Touch'n Go, GCash 등)으로 직접 결제 가능

이 모델이 **전국으로 확대되는 흐름**에 맞춰, EasyToGoKorea가 **QR 결제 허브** 역할을 할 수 있습니다.

### 정부 로드맵

```
2024  제주 QR 버스 도입 (제로페이 + AlipayPlus)
2025  제로페이 외국인 QR 결제 MOU (금융위원회 + 한국간편결제진흥원)
      → "국경 간 소액결제 정산 허브" 구축
2026  개방형 대중교통 결제 연구 용역 (국토부 + 서울시)
2030  Open-loop Transit 완성 목표
      → 해외 신용카드로 직접 버스/지하철 탑승
```

## 💡 3. EasyToGoKorea의 접근 전략

> **직접 결제 처리는 하지 않고, "교통 결제 안내 + QR 허브 + 정보 통합" 역할에 집중**

### 전략 A: 정보 통합 허브 (즉시 구현 가능)

앱이 직접 결제를 처리하지 않더라도 **교통 정보 + 결제 수단 안내**만으로 큰 가치 제공:

| 기능 | 설명 |
|------|------|
| **결제 수단 추천** | 사용자 국적·체류 기간·방문 도시 기반으로 최적 교통카드 추천 |
| **실시간 노선 검색** | 버스/지하철 실시간 도착 정보 + 영문 안내 |
| **요금 계산기** | 출발지→도착지 교통비 자동 계산 (환율 포함, 본국 통화 표시) |
| **충전소 찾기** | 현재 위치 근처 티머니 충전 가능 편의점 표시 |
| **QR 결제 지원 노선 표시** | 제주 등 QR 직접 결제 가능 노선/역 하이라이팅 |

### 전략 B: 제로페이 QR 연동 (파트너십 필요)

제로페이/AlipayPlus와 파트너십 체결 시:

| 기능 | 설명 |
|------|------|
| **앱 내 QR 생성** | EasyToGoKorea 앱에서 직접 교통 결제용 QR 생성 |
| **해외 결제수단 연결** | 사용자의 해외 카드/결제앱 연동 → QR 자동 생성 |
| **탑승 기록 통합** | 교통 이용 내역을 일정 플래너에 자동 반영 |

> ⚠️ **제로페이 API는 공개 개발자 문서가 없으며, 파트너사 계약을 통해서만 연동 가능**합니다. 한국간편결제진흥원에 사업 제안이 필요합니다.

### 전략 C: 선불형 디지털 교통카드 (장기 목표)

Open-loop Transit 도입(~2030) 이후:

| 기능 | 설명 |
|------|------|
| **앱 내 가상 교통카드** | EMV 기반 가상 교통카드 발급 |
| **해외 카드로 충전** | Visa/MasterCard 해외 결제 직접 충전 |
| **NFC/QR 이중 지원** | NFC 지원 기기는 태그, 미지원 기기는 QR |

## 📱 5. MVP 기능 설계 (전략 A — 즉시 구현)

### 화면 흐름

```
┌─────────────────────────────┐
│  🚇 교통 (Transport)         │
├─────────────────────────────┤
│                             │
│  ┌─────────┐ ┌─────────┐   │
│  │ 🗺️ 경로  │ │ 💳 카드  │   │
│  │  검색    │ │  추천    │   │
│  └─────────┘ └─────────┘   │
│  ┌─────────┐ ┌─────────┐   │
│  │ 💰 요금  │ │ 🔍 충전소 │   │
│  │  계산기  │ │  찾기    │   │
│  └─────────┘ └─────────┘   │
│                             │
│  ── 실시간 주변 교통 ──      │
│  🚌 버스 301  → 3분 후 도착  │
│  🚇 2호선    → 2분 후 도착   │
│  🚌 버스 740  → 7분 후 도착  │
│                             │
└─────────────────────────────┘
```

### 교통카드 추천 로직

```
입력: 국적, 체류 기간, 방문 도시, 소지 기기(iOS/Android)

IF 체류 ≤ 3일 AND 서울만:
    → 기후동행카드 단기권 추천

IF 체류 ≤ 7일 AND 전국:
    → WOWPASS 추천 (환전+교통 통합)

IF 체류 > 7일:
    → 티머니 카드 추천 (범용성)

IF iPhone 사용자:
    → Apple Pay 티머니 보조 안내

IF 국적 ∈ {중국, 태국, 말레이시아, 싱가포르, ...}:
    → 제주 방문 시 본국 QR앱 직접 사용 안내

IF KTX 이용 예정:
    → 코레일패스 플러스 추가 추천
```
