## 1. "자체 기술 부재" 우려 대응

> **핵심 논리**: "기술 발명이 아니라, 기술 통합(Integration)의 혁신"

### 🔬 독자적 알고리즘 (남의 기술이 아닌 자체 설계)

| 자체 기술 | 설명 | 경쟁사 대비 |
|-----------|------|-------------|
| **Vote-to-Scale 공간 랭킹** | GeoHash + 감쇠함수 + 언어 친화도 결합 | 현존 경쟁 기능 없음 |
| **4-Tier 하이브리드 실내 측위** | Camera→BLE→IMU→QR 4단 폴백 | 네이버(Visual만), Google(VPS만) 대비 우위 |
| **No-Storage 의료 파이프라인** | On-device AI → OCR → PDF → 네이티브 공유만 | GDPR/PIPA 동시 준수 의료관광 앱 없음 |
| **Decay 기반 콘텐츠 생명주기** | `score = (votes^0.8) * decay(age) * lang_bonus * dist_weight` | 자체 설계 공식 |

## 2. 예상 질의 응답 (Q&A)

> **Q: "Google ARCore를 쓰면 구글이 같은 걸 만들면 끝 아닌가?"**
> A: Google은 플랫폼 제공자이지, 한국 관광 특화 서비스 사업자가 아닙니다.
> ARCore는 "시멘트", EasyToGo는 그 시멘트로 짓는 "건물"입니다.

> **Q: "네이버가 실내 AR 내비를 이미 하고 있잖아?"**
> A: (1) API 비공개 (2) 한국어 전용 (3) 의료+교통+AR 수직 통합 없음

> **Q: "혼자서 이걸 다 만들 수 있나?"**
> A: 전부 직접 만드는 것이 아닙니다. API는 연동, AI는 활용, 자체 개발은 Rust 코어 + AR 알고리즘 + Tauri 아키텍처에 집중.
