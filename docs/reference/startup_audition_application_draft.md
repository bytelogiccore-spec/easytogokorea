## 1. 문제의식 — 왜 이 서비스가 필요한가?

### 🌏 한국은 미래 도시의 인프라를 갖추고 있지만, 외국인은 체감하지 못합니다

한국은 세계 1위 5G 보급률, 전국 QR 인프라(전봇대·가로등), 98% 카드 결제율, K-의료 경쟁력을 갖춘 **첨단 도시 국가**입니다. 그러나 정작 한국을 찾는 **2,500만 외국인 관광객**은 여전히:

| 불편 | 현실 |
|------|------|
| **메뉴판이 한국어** | 식당 앞에서 망설이다 돌아감 → 소상공인 매출 손실 |
| **예약 불가** | 전화 예약은 한국어만 가능, 온라인 예약 없는 가게가 대다수 |
| **정보 파편화** | 교통·맛집·축제·병원 정보가 각각 다른 앱에 흩어져 있음 |
| **의료 접근 장벽** | 한국 의료가 세계적이지만, 외국인은 병원 찾기·보험 청구·통역이 모두 어려움 |
| **디지털 소통 부재** | 현장의 감동을 공유할 디지털 소통 창구가 없음 |

동시에, 이들을 맞이하는 **소상공인 770만 개**도:

| 고민 | 현실 |
|------|------|
| **외국인 손님 받고 싶지만** | 다국어 메뉴판 제작 비용 부담, 번역 업데이트 어려움 |
| **예약 시스템 도입하고 싶지만** | 네이버·배민 수수료 높고, 월 고정비 부담 |
| **자체 서버 운영 기술 없음** | IT 인력 없어 플랫폼에 종속될 수밖에 없음 |

> **핵심**: 한국의 첨단 인프라와 외국인의 실제 경험 사이에 **거대한 갭**이 존재합니다. EasyToGo는 이 갭을 세 가지로 메웁니다:

> 1. **AR 레이어** — 카메라를 들면 간판·메뉴·지하철이 다국어 / AR로 길안내
> 2. **자체 호스팅 SaaS** — 소상공인이 수수료 없이 외국인 예약·주문을 받음 / 결제 간편화 / QR코드 다국어 메뉴판
> 3. **의료관광 자동화** — 병원 찾기→통역→보험 청구까지 앱 하나로 완결

### 🔥 핵심 기술 경쟁력: 투트랙(Two-Track) 의료 API 전략 & 제로 스토리지

글로벌 의료 정보 표준(FHIR)과 한국의 공공 PHR(마이헬스웨이)이 외국인 관광객에게 적용되지 않는 치명적 한계를 **독자적인 기술 아키텍처**로 완벽히 극복했습니다.

**1. Inbound (외국인 환자 방한) 👉 프라이빗 EMR 연동 (No-Friction B2B)**
- **한계점 (경쟁사 기각 사유)**: 외국인은 한국 주민등록번호가 없어 보건복지부의 '마이헬스웨이' 공공 API를 통한 데이터 수집이 원천적으로 불가능합니다.
- **EasyToGo의 솔루션 (플랜 B)**: 병원 원무과 PC에 **EasyToGo Partner 앱(RPA 에이전트)**을 설치하거나, 유비케어 등 민간 EMR API와 직접 연동합니다. 의사가 기존 하던 대로 한국어로 차팅하고 [저장]하면, 환자 앱으로 영문 변환된 수술 기록 및 처방전이 1초 만에 자동 동기화됩니다. (병원 측 추가 업무 0%)

**3. Zero-Storage (서버 무저장) 온디바이스 암호화**
- **문제점 / 법적 규제**: 민감한 환자 진료 데이터를 외부 클라우드 서버에 평문 저장하는 것은 의료법 및 글로벌 개인정보보호법(HIPAA/GDPR)에 위배됩니다.
- **EasyToGo의 보안 설계**: 병원의 차팅 데이터는 환자의 폰(**온디바이스**) 안에서만 영문 PDF로 렌더링되고 즉시 암호화됩니다. 당사 서버는 내용물을 볼 수 없는 '잠긴 상자'만 보관하며, 병원-환자 간 P2P로만 열쇠가 교환되어 **법적 리스크를 원천 차단**합니다.

**4. 비즈니스 리스크 방어: 글로벌 사보험 연동 전략 (Pay & Claim)**
- **문제점 (Direct Billing의 한계)**: 대다수 의료 벤처가 글로벌 보험사(Bupa, Cigna)에 진료비를 직접 청구하려는 TPA(제3자 관리) 모델을 시도하다 막대한 지급 보증금과 법적 책임(의료 소송)에 부딪혀 폐업합니다. 또한 한국의 질병코드(KCD)와 글로벌 코드(ICD-10)가 달라 청구가 거절됩니다.
- **EasyToGo의 초격차 아키텍처 (B2B SaaS 큐 및 비식별화 매핑)**: 당사는 위험한 직접 청구를 대행하지 않습니다. 대신 환자가 선결제(Pay)를 마치면, 병원 클라이언트에서 **환자 식별 정보(PHI)를 토큰화하여 원천 차단**한 순수 한국어 차트만 당사 서버에 비동기(Queue)로 전송합니다. 서버의 LLM 및 고성능 Rust 워커 노드는 KCD를 추출하고 타겟 국가(`target_country`) 파라미터에 맞춰 강력하게 다중 라우팅(Dynamic Routing)하여 전송합니다.
  - **🇺🇸 미국 타겟**: Rust 엔진으로 CPT 코드를 고속 매핑 후 미국 표준 `HCFA-1500` 폼 반환
  - **🇪🇺/🇨🇳 유럽·중국 타겟**: 글로벌 사보험사(Allianz 등) 맞춤형 `Itemized Bill(상세 영문 영수증)` 반환
  - **🇯🇵 일본 타겟**: 일본 법정 규격인 해외요양비 전용 `진료내용명세서(A4양식)` 자동 번역 반환
  서버에서 반환받은 코드는 병원 로컬 클라이언트에서 환자 정보를 다시 덧씌워(Re-identification) 완벽한 영문/다국어 '진료비 영수증(Superbill) PDF'로 렌더링되고, 환자의 스마트폰에 3초 만에 꽂아줍니다. 환자는 이 서류를 탭 한 번으로 본국 보험사에 제출(Claim)만 하면 됩니다.
- **Tax Refund 및 Warranty 패스 (성형/피부과 대응)**: 사보험이 적용되지 않는 미용 성형 분야(전체 의료관광의 70%)는 앱 내에서 부가세 즉시 환급(Tax Refund) 모듈이나 위·변조 불가 '글로벌 수술 보증서(NFT)' 발급으로 우회하여 압도적인 소비자 신뢰를 창출합니다.

**5. 압도적 마진의 B2C 과금 전환 전략 (Friction Zero)**
- **문제점 (기존 SaaS의 원가 부담)**: 클리어링하우스 API(예: 자격 조회, EDI 전송) 사용 시 보험사 1회 연동마다 건당 약 $0.15~$0.25의 원가가 발생하며, 이를 병원 SaaS 구독료에 전가하면 영업 허들(Friction)이 급격히 높아집니다.
- **EasyToGo의 해결책 (환자 대상 In-app 결제)**: 이 서류 생성 단계에서 원가 부담을 병원에 묻지 않고 철저히 "무료"화시킵니다. 대신 앱을 사용하는 **외국인 환자(C)**가 "현지 보험 적용 여부 실시간 확인" 및 "완벽한 영문 서류 다운로드"라는 확실한 편익을 위해 앱 내에서 $2.99를 결제하도록 설계했습니다. 
- **효과**: 병원은 비용 저항 없이 스위치를 켜게 되고, 플랫폼은 API 원가 대비 1,000% 이상의 마진을 창출하는 폭발적인 Cash Cow가 완성됩니다.

### ⚔️ 글로벌 경쟁 우위: 공간적/인적 인프라 한계의 IT 혁신

아시아 의료관광 시장의 패권 경쟁에서 한국의 불리한 포지셔닝을 **플랫폼 기술력**으로 단숨에 역전시킵니다.

- **인도 (언어적 우위)**: 의사 전원이 영어로 진료하며, EMR 차트와 처방전이 병원 내에서부터 영문으로 직접 작성됨. (IT 번역 기술 및 통역사 불필요)
- **태국/베트남 (자본·하드웨어 우위)**: 수백억 원을 들여 다국어 편의가 갖춰진 초대형 병원(범룽랏 등)을 건립하고, 각국의 언어를 구사하는 코디네이터를 수십 명씩 정규 채용하여 1:1 전담 마크함.
- **한국의 한계**: K-의료의 수준(성형, 피부, 중증)은 세계 최고이나, 동네 의원 단위가 주축이라 전담 시스템과 다국어 인력을 별도 구축할 자본이 없고 차팅도 100% 한글의 혼합(Free-text) 방식에 머물러 있음.
