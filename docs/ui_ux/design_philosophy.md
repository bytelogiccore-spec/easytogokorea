# EasyToGo UI Design Philosophy & Recommendation Engine 🦢

> **핵심 철학**: *"수면 위(UI)는 고요하고 우아하게, 수면 아래(AI)는 치열하고 치밀하게"*
> **최종 수정**: 2026-03-09

---

## 1. 극단적 미니멀리즘 (Extreme Minimalism)

우리의 UI는 일반적인 여행 앱처럼 버튼과 배너를 빽빽하게 나열하지 않습니다. 외국인 관광객의 인지 부하를 0으로 만드는 것이 목표입니다.

*   **콘텐츠가 곧 UI (Content as UI)**
    *   박스, 선, 데코레이션 요소를 철저히 배제합니다.
    *   여백과 압도적인 텍스트 크기(타이포그래피)만으로 위계를 나눕니다.
    *   여행 추천 비디오 등의 리치 미디어는 화면 전체(100%)를 채우며, 조작 요소는 보일 듯 말 듯 투명하게 얹힙니다. (TikTok, Reels 스타일)
*   **선택의 제거 (Elimination of Choice)**
    *   사용자에게 카테고리를 고르라고 강요하지 않습니다.
    *   홈 화면은 거대한 `[Search or Scan]` 입력창과 최소한의 방향성(식당, 교통)만 제시합니다.

---

## 2. 제로 클릭 추천 시스템 (Zero-Click Recommendation Engine)

미니멀리즘 UI가 성립하려면 앱이 알아서 **"지금 이 사람에게 가장 필요한 딱 1개의 정보"**를 선별해 던져주어야 합니다. UI에서 덜어낸 복잡함은 백엔드의 추천 엔진이 대신 감당합니다.

### 1) "수동 입력 없는" 데이터 수집 (Implicit Data)
외국인 관광객은 회원가입이나 긴 취향 설문을 싫어합니다. 앱 설치 즉시 발생하는 환경 데이터만으로 초기 추천을 완성합니다.

| 데이터 소스 | 활용 예시 (컨텍스트) |
| :--- | :--- |
| **GPS 위치** | 현재 홍대입구역 → "근처 힙한 카페 AR 리뷰 숏폼" 1순위 노출 |
| **현재 시간** | 저녁 6시 30분 → 식당 리스트 우선 노출 / 자정 → 교통(심야버스/택시) 우선 노출 |
| **스마트폰 언어/국적** | 일본어 폰 → 일본인 관광객이 어제 가장 많이 스캔한 성수동 팝업스토어 노출 |
| **체류 기간 (패턴)** | 앱 최초 구동(공항) → 공항버스 타는 법 / 4일 차 → 귀국 전 쇼핑 리스트 |

### 2) 상호작용 기반 행동 데이터 (Interaction Data)
사용자가 앱을 쓰면서 남기는 잔향(Trace)을 실시간으로 분석합니다.

*   **카메라 스캔 기록**: 어제 '올리브영' 라벨을 AR로 3번 스캔했다면 → 다음 날 뷰티 팝업스토어나 K-뷰티 할인 쿠폰 비디오를 우선 추천합니다.
*   **영상 체류 시간 (Watch Time)**: 삼겹살 비디오는 3초 만에 넘기고, 비건/사찰음식 비디오를 15초 끝까지 봤다면 → 채식주의자로 분류하여 이후 추천에 반영합니다.

---

## 3. 작동 시나리오 "백조의 발차기 전략"

1.  **관광객 액션**: 아무 생각 없이 앱을 켜고 `[Restaurants →]` 라는 거대한 텍스트를 누릅니다.
2.  **백엔드 로직 (수면 아래)**:
    *   Rust 코어의 추천 엔진이 0.1초 만에 계산합니다.
    *   *조건: [지금 시간 저녁 7시] + [위치 명동] + [과거 스캔 기록=길거리 음식점] + [미국인]*
3.  **UI 결과 (수면 위)**:
    *   복잡한 리스트 대신, 명동 길거리 음식점 숏폼 비디오가 화면 가득(Full-screen) 뜹니다.
    *   사용자는 위아래로 스와이프하며 영상을 즐기다, 마음에 들면 `[AR 안내 시작]` 버튼만 누르면 됩니다.

> **결론**: EasyToGo의 미니멀리즘은 장식이 아니라 **"고도화된 AI 추천을 담아내기 위한 가장 완벽한 그릇"**입니다.
