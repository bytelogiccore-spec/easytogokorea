## 1. 제안 배경

### 정부 정책 목표와 현실의 간극

한국보건산업진흥원(KHIDI)의 「제2차 의료 해외진출 및 외국인환자 유치 지원 종합계획(2022-2026)」에 따르면, **2026년까지 외국인 환자 50만 명 유치**가 핵심 목표입니다.

그러나 현장에서 외국인 환자가 겪는 3대 진입 장벽이 이 목표 달성을 저해하고 있습니다:

| # | 장벽 | 현재 상황 |
|---|------|-----------|
| 1 | **정보 파편화** | 신뢰 가능한 병원·에이전시 정보가 10개 이상 플랫폼에 분산 |
| 2 | **언어 장벽** | 의료 접수·동의서·처방전이 한국어 전용. 현장 통역 부재 빈번 |
| 3 | **보험 처리 복잡성** | 해외 보험 청구 절차가 수작업 기반, 환자가 포기하는 경우 다수 |

### 민간 플랫폼의 한계

기존 의료관광 플랫폼(메디컬코리아, 각 병원 자체 사이트)은 **정보 제공에 집중**되어 있으며,
실제 예약-통역-보험 청구까지의 **End-to-End 디지털 자동화**가 부재합니다.

## 2. 제안 솔루션: EasyToGo 의료관광 모듈

### 핵심 가치: "발견-예약-통역-청구" 4단계 원스톱 자동화

```
[환자 여정]

① 발견     → 다국어 병원·에이전시 마켓플레이스
               (KTO 메타데이터 표준 채택)

② 예약     → 앱 내 다국어 예약 + PG 연동 보증금 결제

③ 통역     → On-device AI 실시간 의료 통역
               (서버 미전송 → 개인의료정보 보호)

④ 보험청구 → 영수증 OCR → AI 구조화 → PDF 생성
               → 원터치 보험사 이메일 전송
```

### 개인의료정보 보호: "No-Storage" 아키텍처

| 구분 | 기존 플랫폼 | EasyToGo |
|------|-------------|----------|
| 의료 데이터 처리 | 서버 업로드·저장 | **기기 내 AI로만 처리** |
| 데이터 잔류 | 사업자 서버에 잔류 | **처리 후 즉시 삭제** |
| 정보 전달 방식 | 사업자 경유 전달 | **환자가 직접 공유** |
| 관련 법규 준수 | 별도 동의 필요 | **구조적으로 준수** |

> 이 설계는 「개인정보보호법」 및 「의료법」 상 민감정보 처리 규정을 
> **기술 아키텍처 수준에서** 준수합니다.

## 3. B2B2C 마켓플레이스 구조

### 3자 가치 체인

| 참여자 | 역할 | 받는 가치 |
|--------|------|-----------|
| **외국인 환자** | 이용자 | 원스톱 병원 발견·예약·통역·보험 청구 |
| **의료 에이전시** | 파트너 | SaaS 기반 디지털 매장 + 예약 관리 + 글로벌 노출 |
| **병원/의원** | 공급자 | 다국어 디지털 접수 + 검증된 에이전시 리드 |

### 수익 구조

| 수익원 | 대상 | 예상 단가 |
|--------|------|-----------|
| 월 SaaS 이용료 | 병원/의원 | 10만 원~/월 |
| 월 SaaS + 수수료 | 에이전시 | 20만 원~/월 + 예약 수수료 5~10% |
| 광고 | 병원/브랜드 | AR 기반 위치 노출 (CPC/CPM) |

## 4. KHIDI 사업과의 시너지

### 「제2차 종합계획」 5대 전략별 연계

| KHIDI 전략 | EasyToGo 기여 |
|------------|---------------|
| ① 의료한류 확산 | 다국어 앱을 통한 한국 의료 글로벌 노출 확대 |
| ② 진료영역 다각화 | 성형·피부 외에 검진·정형·치과 등 마켓플레이스 커버리지 확대 |
| ③ ICT 기반 사전·사후관리 | On-device AI 통역 + 보험 청구 자동화 = 사후관리 강화 |
| ④ 국제교류·협력 확대 | 다국어 플랫폼으로 해외 보험사·기관과 연계 가능 |
| ⑤ 외국인환자 안전·질 관리 | 에이전시 인증 시스템 + 환자 리뷰 다국어 공개 |

- KHIDI 주관 해외 로드쇼 시 **디지털 도구로서 동반 참여** 제안
