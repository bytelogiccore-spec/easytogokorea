# EasyToGoKorea UI 디자인 가이드

## 1. 목적
- **통일성**: 모든 화면에서 헤더·버튼·카드 스타일을 일관되게 유지
- **간소화**: 한 화면의 시각적 요소 수를 줄여 인지 부담 감소

---

## 2. 통일성 규칙

### 2.1 헤더 (공통)
- **서브 페이지**(뒤로가기 있는 화면): `page-header` 사용
  - 왼쪽: 뒤로가기 버튼 (`icon-btn clear`)
  - 가운데: 제목 (`header-title`, 한 줄)
  - 오른쪽: 선택(저장/장바구니 등) 또는 빈 칸
- **인라인 스타일 금지**: 헤더 배경·패딩은 `styles.css`의 `.page-header`만 사용
- **status-bar**: 기본 배경. 특수 화면(맵·풀스크린)만 `.status-bar.transparent` 사용

### 2.2 필터/탭 칩
- **클래스**: `filter-chip`, `filter-chip.active`
- **스타일**: `styles.css`의 `.filter-chip` 정의만 사용 (페이지별 `<style>` 내 재정의 지양)
- **개수**: 한 줄에 4~5개 이하, 필요 시 가로 스크롤

### 2.3 카드
- **기본**: `.card` + 필요 시 `.feed-card`, `.drag-card` 등 보조 클래스
- **둥근 모서리**: `var(--radius-md)` 또는 `var(--radius-lg)`만 사용
- **그림자**: `var(--shadow-soft)` / 호버 시 `var(--shadow-hover)`

### 2.4 버튼
- **주요 액션**: `pill-btn primary` 또는 `pill-btn full-width primary`
- **보조 액션**: `pill-btn mild` 또는 `icon-btn`
- **인라인 색/배경**: 가급적 사용하지 말고, 공통 클래스로 처리

### 2.5 폰트
- **Pretendard**: `400, 500, 600, 700, 800` 한 번만 로드 (동일 링크)
- **제목**: `font-weight: 700` 또는 `800`, **본문**: `600` 이하

---

## 3. 요소 간소화 규칙

### 3.1 한 화면당 블록 수
- **메인(홈)**: 그리팅 + 카드 2개(날씨/지갑) + 섹션 2~3개(예: 팝업 이벤트, 추천, 빠른 실행)
- **리스트 페이지**: 헤더 + 필터 1줄 + 카드 리스트 (카드당 정보는 아래 기준)

### 3.2 카드당 정보량
- **리스트 카드**: 제목 + 부가 정보 1~2줄 + (선택) 호스트/가격 한 줄
- **상세 카드**: 섹션별로 나누고, 한 섹션당 4개 이하 항목 권장

### 3.3 빠른 실행/액션
- **4개**: 2x2 그리드 유지
- **5개 이상**: 4개만 노출 + "더보기" 또는 두 번째 줄로 이탈

### 3.4 배너/공지
- 한 화면에 **상단 배너 1개**만 (예: 무료배송 안내). 두 번째는 접기 또는 제거

## 4. 적용 체크리스트 (페이지 수정 시)
- [ ] 헤더가 `page-header` 또는 공통 `.header` + `.header-center` 구조인가?
- [ ] status-bar에 인라인 스타일이 없는가?
- [ ] filter-chip을 페이지 `<style>`에서 다시 정의하지 않았는가?
- [ ] 카드/버튼에 인라인 `border`, `background`를 최소화했는가?
- [ ] 한 화면에 섹션/카드 수가 가이드(3~4 블록)를 넘지 않는가?
